## 二、六度模型照见

### 【有无】AI时代什么存在了、什么消失了？

**存在了的：**
- AI让"建造"门槛归零——任何人可以用自然语言生成代码
- 快速原型能力——从月到小时的压缩
- "行业老炮+AI"组合——不需要CS学位也能构建系统

**消失了的：**
- "我会写代码"的护城河——人人都会的时代
- 建造速度本身就是优势——很快，但不是护城河
- 用"我有多努力"来感动自己——勤奋不等于有效

**最深的认知层：** AI时代"建造能力"从稀缺变成冗余，但"验证能力"（验证需求、验证PMF、验证方向）从未如此稀缺。

---

### 【难易】什么是真正难的？

| 过去认为难的 | 现在真正难的 |
|------------|-------------|
| 写代码/搭系统 | 验证这是不是用户要的东西 |
| 快速迭代 | 判断该往哪个方向迭代 |
| 技术实现 | 抵抗confirmation bias——让AI不顺着你说 |
| 功能开发 | 识别什么是噪音什么是信号 |

**最深的认知层：** AI把"执行之难"消解了，但"判断之难"被放大十倍。做一个没人要的东西，速度越快，损失越大。

---

### 【长短】时间尺度上什么变了？

**变了：**
- 原型时间：月→小时（质的压缩）
- 代码产出：行数不再代表价值
- 迭代周期：加速但不是缩短验证周期

**没变：**
- 用户验证时间——人没变，决策周期没变
- PMF确认时间——仍需真实用户反馈
- 认知偏见——AI加剧而非减轻

**最深的认知层：** "刹车片"被拆了（原型更快），但"刹车距离"没变（验证）。结果：冲进悬崖的速度更快了。

---

### 【高下】Founder该站在哪？

**从键盘上移到：**
1. **方向盘后**——决定去哪，不是怎么踩油门
2. **观察哨**——识别真假PMF，不是埋头编码
3. **过滤层**——挡住噪音，让团队专注

**从执行者变为：**
- 判断力 > 执行力
- 方向感 > 技术能力
- 过滤噪音 > 产生噪音

**最深的认知层：** Founder的最高位置是"过滤器"，不是"生产者"。你过滤什么，决定了系统产出什么。

---

### 【音声】什么信号是真PMF？什么是噪音？

**真PMF信号：**
1. Sean Ellis测试：>40%用户"非常失望"如果产品没了
2. 努力曲线转折点：从"推→拉"——用户主动来找你
3. 用户自己讲出使用场景——不是你在问，是他们在说
4. 口碑传播——没营销用户自己分享

**噪音：**
1. "这个很酷"——不是购买
2. "我可以用它来做XXX"——不是"我每天用"
3. 早期粉丝的鼓励——不等于市场验证
4. 数量有限的positive feedback——没有对比就没有伤害

**最深的认知层：** AI会放大你的偏见，让你更容易听到你想听的声音。你需要主动"泼冷水"的机制，而不是只会说"好的"的AI。

---

### 【先后】先验证还是先建造？

**顺序错误：**
```
❌ 建造 → 验证 → 发现没人要 → 重新建造（浪费）
```

**顺序正确：**
```
✅ 验证方向 → 小原型验证 → PMF确认 → 加速建造
```

**最深的认知层：** AI让建造变快，但验证必须前置。在AI时代，"先建造再验证"是最高成本的错误，因为你的刹车片没了。

**"难"是内核。** 所有8条都是一件事：**创始人对抗"本能"的修行。**

- 对抗"想干更多事"的本能 → 去专注瓶颈
- 对抗"用复杂掩盖无力"的本能 → 去追求简洁
- 对抗"害怕错过"的本能 → 去过滤噪音
- 对抗"担心不被喜欢"的本能 → 去直接说"不"

**最深的难，就是最深的认知：创业不是"做加法"，而是"做减法"；管理者不是"成就者"，而是"过滤器"。**
